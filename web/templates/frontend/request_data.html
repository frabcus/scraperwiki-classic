ScraperWiki is a place where raw information is transformed into living data

Please select the option on the right that best describes your needs…

I need some data that I want to keep private

We get the data and store it for you. The cost will depend on the size and complexity of the information being gathered. It can range from a couple of hours to a few days in duration. When we know what you want, we will give you a quote. We pay our developer community to gather the data.

I need data and I also want a visualisation

This is a combination of getting the data, storing it, and creating a visualisation that tells a story. We usually ask you to create a short brief with an idea of what you want to do and also a link to the data that you are proposing to mine. We will help you to do this directly or with a ScraperWiki partner.

I need a data application for my website

Data applications are designed to drive continuous traffic to a website and also offer the possibility of generating revenue through a subscription model for services like email alerts. This is an automated service.

I want to merge content from systems that cannot talk to each other (ETL) – Help!

Sometimes it can be easier to scrape data from systems than to re-engineer the back-end. In this context ScraperWiki acts as a data hub for the purposes of transforming data into a central datastore. It is a more cost effective ‘ETL’ – Extract, Transform and Load and a flexible option for building an API for web or mobile applications.

I need some data and I want to make it public & open

This is a free service, if you have the technical skills just go and do it! If not, we can broadcast this request to our developer community. Alternatively if you have budget we can pay a developer to do it for you. The cost can range in price and will depend on the size and complexity of the information.
