External Python libraries supported in ScraperWiki

ScraperWiki supports a number of external Python libraries that we recommend for screen scraping, data analysis and data visualisation.

If you would like us to add a library that isn't listed here, please get in touch.

  • urllib2, urlparse Standard Python libraries for opening URLs. docs
  • lxml Parse HTML with a specialist screen scraping library. docs
  • BeautifulSoup Parse HTML with an alternative screen scraping library. Relatively quick to learn, but no longer supported. docs
  • pyquery Make jquery like queries from Python. docs
  • mechanize Navigate and complete HTML forms. docs
  • selenium Automate operating real browsers like Firefox. NB: Only useful in ScraperWiki if you have a Selenium server to point it to. docs
  • xlrd, xlwt, xlutils Read, write and process Excel files. docs
  • PDFMiner Python PDF parser and analyzer. docs
  • pyPdf Manipulate PDF files. docs
  • geopy Converts addresses into latitude/longitude, and measures distances on the earth. docs
  • Python YQL Yahoo Query Language, an expressive SQL-like language that lets you query, filter, and join data across Web services. docs
  • pygooglechart Generate charts using the Google Chart API. docs
  • Python Imaging Library (PIL) Image processing, lots of file formats. docs
  • iCalendar Prase and generate calendar files. docs
  • RDFLib Input and output linked data triples as RDF. docs
  • NumPy Large, multi-dimensional arrays, matrices and functions to operate on them. docs
  • RPy Call out to GNU R, a popular statistical computing and graphics package. docs
  • Natural Language Toolkit (NLTK) Natural language processing and text analytics. docs
  • FOM (Fluid Object Mapper) Read or write to the Fluidinfo shared database. docs
